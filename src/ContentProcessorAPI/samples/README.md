# Sample claim packs — DEMO / SYNTHETIC DATA

> **All data in this folder is fictional and was generated for demo and
> testing purposes only.** Names, addresses, phone numbers, license plates,
> VINs, claim numbers, policy numbers, monetary amounts, dates, and any
> photos are synthetic. No real customer, member, vehicle, or incident is
> represented. Do not treat any extracted field as a real-world reference.

Each subfolder contains a multi-document pack that exercises the
**Auto Claim** schema set (`AutoInsuranceClaimForm`,
`PoliceReportDocument`, `RepairEstimateDocument`,
`DamagedVehicleImageAssessment`).

| Folder                                | Scenario                                     |
| ------------------------------------- | -------------------------------------------- |
| `claim_collision_commercial_hint/`    | Personal-use policy with subtle TNC/ride-share usage cues — exercises gap-analysis. |
| `claim_demo_persona/`                 | Persona document pack referenced by the Claims Demo journey UI. |
| `claim_hail_storm/`                   | Comprehensive (weather) loss; multi-photo damage. |
| `claim_theft_vandalism/`              | Recovered-after-theft scenario; non-collision. |
| `schemas/`                            | JSON schemas registered by post-deployment for the Auto Claim set. |

PDF and image fixtures are regenerated by the per-folder
`generate_pdfs.py` / `generate_image.py` scripts. Re-running those scripts
produces deterministic but still entirely synthetic content — never seed
them with real customer data.

For upload helpers, see `upload_files.ps1` / `upload_files.sh` in this
folder.
